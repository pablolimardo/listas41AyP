Listas en Python

Una lista guarda varios valores en una sola variable. Como un registro de lecturas de la represa.

📋
Fundamento
¿Qué es una lista?
🗃️
Una lista es como una planilla de registros: tiene celdas numeradas (desde la 0), y en cada celda podés guardar un valor. Podés agregar, quitar, buscar y ordenar los valores.
# Crear una lista — los valores van entre corchetes niveles = [42.3, 45.1, 44.8, 47.2, 46.0, 48.5] # Acceso por índice — el primero es índice 0 print(niveles[0]) # → 42.3 (primer elemento) print(niveles[2]) # → 44.8 (tercer elemento) print(niveles[-1]) # → 48.5 (último elemento) print(niveles[-2]) # → 46.0 (penúltimo) # Slicing — tomar una porción print(niveles[1:4]) # → [45.1, 44.8, 47.2] (índice 1, 2, 3) print(niveles[:3]) # → [42.3, 45.1, 44.8] (desde el inicio) print(niveles[3:]) # → [47.2, 46.0, 48.5] (desde índice 3) # Largo de la lista print(len(niveles)) # → 6
Los índices empiezan en 0, no en 1. Una lista de 6 elementos tiene índices del 0 al 5. El índice 6 da error (IndexError). Con índices negativos: -1 es el último, -2 el penúltimo.

for con listas

El for es la herramienta para recorrer y analizar los datos de una lista.

🔁
Tres formas de recorrer
for, for + enumerate, for + range(len())
# Forma 1: directo — cuando solo necesitás el valor niveles = [42.3, 45.1, 44.8, 47.2, 48.5] for nivel in niveles: print(f"Nivel: {nivel} m") # Forma 2: enumerate — cuando necesitás valor Y posición for i, nivel in enumerate(niveles): print(f"Hora {i+1}: {nivel} m") # → Hora 1: 42.3 m | Hora 2: 45.1 m | ... # Forma 3: range(len()) — para modificar elementos de la lista for i in range(len(niveles)): if niveles[i] > 47.0: print(f"⚠ Índice {i}: nivel {niveles[i]} m supera el límite")

Acumuladores con for — calcular máximo, mínimo y promedio:

# Sin usar max(), min(), sum() — algoritmo manual niveles = [42.3, 45.1, 44.8, 47.2, 48.5] maximo = niveles[0] # arrancar con el primer valor minimo = niveles[0] suma = 0 alarmas = 0 for nivel in niveles: if nivel > maximo: maximo = nivel if nivel < minimo: minimo = nivel suma += nivel if nivel > 47.0: alarmas += 1 promedio = suma / len(niveles) print(f"Max: {maximo} | Min: {minimo} | Prom: {promedio:.1f} | Alarmas: {alarmas}")

Pilas — Stack LIFO

LIFO: Last In, First Out. El último que entra es el primero en salir.

📚
Concepto LIFO
¿Qué es una pila y para qué sirve?
🍽️
Imaginá una pila de platos sucios. Siempre ponés el nuevo encima (push) y siempre sacás el de arriba primero (pop). El primero que pusiste es el último en salir. Eso es LIFO.
# Una Pila se implementa con una lista normal historial = [] # pila vacía # push → agregar al tope de la pila historial.append(42.3) # → [42.3] historial.append(45.1) # → [42.3, 45.1] historial.append(48.5) # → [42.3, 45.1, 48.5] # peek → ver el tope sin sacarlo tope = historial[-1] # → 48.5 (sin modificar la pila) # pop → sacar el tope de la pila ultimo = historial.pop() # → devuelve 48.5, pila queda [42.3, 45.1] # ¿Está vacía? if not historial: print("Pila vacía")
Aplicación en la represa: el historial de estados de las compuertas es una pila. Cada vez que una compuerta cambia de estado, se apila el registro. Para deshacer el último cambio, se saca el tope de la pila (pop).

Colas — Queue FIFO

FIFO: First In, First Out. El primero que entra es el primero en salir.

🚶
Concepto FIFO
¿Qué es una cola y para qué sirve?
🏪
Imaginá la fila del supermercado. El primero que llega es el primero en ser atendido. Cuando llegás, te sumás al final (enqueue). El que sale es siempre el del frente (dequeue). Eso es FIFO.
# Una Cola también se implementa con una lista alertas = [] # cola vacía # enqueue → agregar al final de la cola alertas.append("Nivel alto: 48.5 m") # entra al final alertas.append("Caudal excesivo") alertas.append("Sensor B sin respuesta") # peek → ver el frente sin sacarlo siguiente = alertas[0] # → "Nivel alto: 48.5 m" # dequeue → sacar el primero de la cola primera = alertas.pop(0) # → "Nivel alto..." (el frente) # ahora alertas = ["Caudal excesivo", "Sensor B sin respuesta"] # ¿Está vacía? if not alertas: print("Sin alertas pendientes")
Aplicación en la represa: la cola de alertas garantiza que las alertas se atienden en el orden en que llegaron. La primera alarma que entró es la primera en atenderse — ninguna se "cuela" en la fila.
▶ Cola interactiva — alertas de la represa
Alerta a encolar
← frente (primero en salir)      fondo → (último en entrar)
La cola está vacía. Usá enqueue() para agregar alertas.
